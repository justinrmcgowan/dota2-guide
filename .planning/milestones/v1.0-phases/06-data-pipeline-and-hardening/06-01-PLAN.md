    3. Create prismlab/backend/data/refresh.py with:
       - `async def refresh_all_data()` -- the main pipeline function:
         a. Creates an OpenDotaClient with settings.opendota_api_key
         b. Opens an async_session
         c. Fetches heroes from OpenDota, upserts into Hero table (use session.merge for each hero, same field mapping as seed.py)
         d. Fetches items from OpenDota, upserts into Item table (use session.merge, same field mapping as seed.py)
         e. Sets updated_at = datetime.now(timezone.utc) on each upserted record
         f. Commits the session
         g. Logs a DataRefreshLog entry with status="success", hero/item counts
         h. On any exception: logs DataRefreshLog with status="failed" and error_message, re-raises
         i. Use logging throughout (logger = logging.getLogger(__name__))
       - `async def get_last_refresh() -> DataRefreshLog | None` -- queries DataRefreshLog for the most recent successful entry, ordered by completed_at desc, limit 1
       - Important: This runs OUTSIDE a request context (from scheduler), so it must create its own async_session, not use FastAPI's get_db dependency.
       - Important: To call async refresh_all_data from APScheduler (sync scheduler), use a wrapper: `def _scheduled_refresh(): asyncio.run(refresh_all_data())`. BUT since FastAPI runs its own event loop, instead use `asyncio.get_event_loop().create_task(refresh_all_data())` from the scheduler callback is wrong too. The cleanest approach: use APScheduler's AsyncIOScheduler (from apscheduler.schedulers.asyncio) which natively supports async jobs. Add `from apscheduler.schedulers.asyncio import AsyncIOScheduler`.

    4. Create prismlab/backend/api/routes/admin.py:
       - POST /admin/refresh-data -- triggers refresh_all_data(), returns {"status": "started", "message": "Data refresh initiated"}. Use BackgroundTasks from FastAPI to run it non-blocking so the endpoint returns immediately. Return 200 with status.
       - GET /api/data-freshness -- calls get_last_refresh(), returns {"last_refresh": ISO timestamp string or null, "heroes_updated": int, "items_updated": int}. If no refresh log exists, check Hero.updated_at from any hero as fallback (this covers the initial seed).
