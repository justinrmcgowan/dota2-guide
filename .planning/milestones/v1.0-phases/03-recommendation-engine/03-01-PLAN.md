```python
from pydantic import BaseModel, Field

class RecommendRequest(BaseModel):
    hero_id: int
    role: int = Field(ge=1, le=5)  # Pos 1-5
    playstyle: str
    side: str  # "radiant" | "dire"
    lane: str  # "safe" | "off" | "mid"
    lane_opponents: list[int] = Field(default_factory=list, max_length=2)
    allies: list[int] = Field(default_factory=list, max_length=4)

class RuleResult(BaseModel):
    item_id: int
    item_name: str
    reasoning: str
    phase: str  # "starting" | "laning" | "core" | "late_game"
    priority: str  # "core" | "situational" | "luxury"

class ItemRecommendation(BaseModel):
    item_id: int
    item_name: str
    reasoning: str
    priority: str  # "core" | "situational" | "luxury"
    conditions: str | None = None  # For situational decision tree cards

class RecommendPhase(BaseModel):
    phase: str  # "starting" | "laning" | "core" | "late_game" | "situational"
    items: list[ItemRecommendation]
    timing: str | None = None
    gold_budget: int | None = None

class LLMRecommendation(BaseModel):
    """Schema for Claude structured output. Used to generate JSON schema for output_config."""
    phases: list[RecommendPhase]
    overall_strategy: str

class RecommendResponse(BaseModel):
    phases: list[RecommendPhase]
    overall_strategy: str | None = None
    fallback: bool = False
    model: str | None = None
    latency_ms: int | None = None
```

**1. Extend `prismlab/backend/data/opendota_client.py`** — add two new methods to `OpenDotaClient`:

```python
async def fetch_hero_matchups(self, hero_id: int) -> list[dict]:
    """Fetch matchup stats for a hero from /heroes/{hero_id}/matchups.

    Returns list of {"hero_id": int, "games_played": int, "wins": int}.
    """
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"{self.BASE_URL}/heroes/{hero_id}/matchups",
            params=self.params,
            timeout=15.0,
        )
        response.raise_for_status()
        return response.json()

async def fetch_hero_item_popularity(self, hero_id: int) -> dict:
    """Fetch item popularity for a hero from /heroes/{hero_id}/itemPopularity.

    Returns {"start_game_items": {item_id: count}, "early_game_items": {...},
             "mid_game_items": {...}, "late_game_items": {...}}.
    """
    async with httpx.AsyncClient() as client:
        response = await client.get(
            f"{self.BASE_URL}/heroes/{hero_id}/itemPopularity",
            params=self.params,
            timeout=15.0,
        )
        response.raise_for_status()
        return response.json()
```

**2. Create `prismlab/backend/data/matchup_service.py`** with:

- `STALE_THRESHOLD = timedelta(hours=24)` constant
- A module-level `_fetch_locks: dict[str, asyncio.Lock]` for deduplicating concurrent fetches per matchup pair (Pitfall 4 from research)

Function `get_or_fetch_matchup(hero_id: int, opponent_id: int, db: AsyncSession, client: OpenDotaClient) -> MatchupData | None`:
  - Query MatchupData table for (hero_id, opponent_id)
  - If cached and fresh (updated_at within 24h), return it
  - If cached but stale, return it immediately AND schedule `asyncio.create_task(_refresh_matchup(...))` in background
  - If not cached, call `_refresh_matchup(...)` synchronously (first request must wait)
  - Never raise -- catch all exceptions, log, return None

Function `_refresh_matchup(hero_id: int, opponent_id: int, db: AsyncSession, client: OpenDotaClient) -> MatchupData | None`:
  - Acquire lock from `_fetch_locks[f"{hero_id}_{opponent_id}"]` (create if absent)
  - Call `client.fetch_hero_matchups(hero_id)`
  - Find the entry where `entry["hero_id"] == opponent_id`
  - Calculate `win_rate = wins / games_played` (handle division by zero -> 0.5)
  - Upsert into MatchupData table via `db.merge()`
  - Return the MatchupData object

Function `get_hero_item_popularity(hero_id: int, db: AsyncSession, client: OpenDotaClient) -> dict | None`:
  - Call `client.fetch_hero_item_popularity(hero_id)`
  - Return the raw dict (start_game_items, early_game_items, mid_game_items, late_game_items)
  - Catch and log errors, return None on failure

Function `get_relevant_items(hero_id: int, role: int, db: AsyncSession) -> list[dict]`:
  - Query Item table filtering out recipes (is_recipe=False), neutrals (is_neutral=False), zero-cost items
  - Filter by role budget: max_cost = 10000 for role 1-3, 5500 for role 4-5
  - Return list of `{"id": item.id, "name": item.name, "cost": item.cost}` dicts
  - Sort by cost ascending, cap at 50 items
  - This is the item catalog pre-filtering for Claude prompts (per CONTEXT.md: "~40 relevant items per request")
