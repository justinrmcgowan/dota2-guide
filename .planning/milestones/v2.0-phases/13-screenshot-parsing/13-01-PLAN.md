Create a module with two functions using `difflib.SequenceMatcher` from stdlib:

```python
def match_hero_name(vision_name: str, heroes: list) -> tuple[object | None, float]:
```
- Takes a hero name from Vision output and list of Hero ORM objects
- Tries exact case-insensitive match on `hero.localized_name` first (ratio = 1.0)
- Falls back to fuzzy match with SequenceMatcher, threshold > 0.7
- Returns `(hero, match_ratio)` tuple -- the ratio is used for confidence scoring

```python
def match_item_name(vision_name: str, items: list) -> tuple[object | None, float]:
```
- Takes an item name from Vision output and list of Item ORM objects
- Tries exact case-insensitive match on `item.name` first (ratio = 1.0)
- Falls back to fuzzy match, threshold > 0.7
- Returns `(item, match_ratio)` tuple

Both functions should handle None/empty strings gracefully.

Confidence translation logic (can be a helper function `resolve_confidence(vision_confidence: str, match_ratio: float) -> str`):
- vision_confidence == "certain" AND match_ratio >= 0.95 -> "high"
- vision_confidence in ("certain", "likely") AND match_ratio >= 0.85 -> "medium"
- Otherwise (if matched at all) -> "low"
