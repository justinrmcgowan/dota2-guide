    **1. Extend recommendation.ts -- add after RecommendRequest interface:**

    ```typescript
    export interface NeutralItemPick {
      item_name: string;
      reasoning: string;
      rank: number;
    }

    export interface NeutralTierRecommendation {
      tier: number;
      items: NeutralItemPick[];
    }
    ```

    Add neutral_items to RecommendResponse:
    ```typescript
    export interface RecommendResponse {
      phases: RecommendPhase[];
      overall_strategy: string | null;
      fallback: boolean;
      model: string | null;
      latency_ms: number | null;
      neutral_items: NeutralTierRecommendation[];
    }
    ```

    **2. Create NeutralItemSection.tsx:**

    Component receives neutral_items: NeutralTierRecommendation[] as prop.

    Tier timing constants: { 1: "5 min", 2: "15 min", 3: "25 min", 4: "35 min", 5: "60 min" }

    Steam CDN base: "https://cdn.cloudflare.steamstatic.com/apps/dota2/images/dota_react/items"

    Component structure per D-01 (dedicated section below timeline):
    - Outer wrapper: same card style as PhaseCard (bg-bg-secondary rounded-lg p-4 border border-bg-elevated)
    - Section header: "BEST NEUTRAL ITEMS" in spectral cyan (text-cyan-accent text-sm font-bold tracking-wider)
    - Per D-03: show ALL 5 tiers, no progressive reveal
    - For each tier in neutral_items (sorted by tier number):
      - Tier sub-header: "T{tier} ({timing})" in text-gray-400 text-xs font-semibold
      - Per D-02: show 2-3 ranked picks with #1 as best
      - Each pick: item image (24x24, from Steam CDN using item_name converted to internal_name: lowercase, spaces to underscores, apostrophes removed), item name, rank badge (#1/#2/#3)
      - Per D-04: short reasoning text below each item (text-gray-400 text-xs)
      - Per D-05: build-path callouts live in the reasoning text, no separate badges

    Internal name conversion: item_name.toLowerCase().replace(/ /g, "_").replace(/'/g, "")
    Add onError handler on img to hide broken images.

    If neutral_items is empty or undefined, return null.

    Use Tailwind classes consistent with PhaseCard:
    - bg-bg-secondary for outer card
    - bg-bg-elevated/50 for tier sub-sections
    - text-cyan-accent for section header
    - text-gray-300 for item names
    - text-gray-400 for reasoning text
