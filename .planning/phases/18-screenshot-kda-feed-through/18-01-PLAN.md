<objective>
Feed parsed enemy KDA and level data from screenshots into Claude's recommendation reasoning.

After this plan, Claude's recommendation reasoning references enemy economic state (e.g., "Enemy PA is 8-1-3 and snowballing -- prioritize defensive items") when screenshot data is available.

Output: End-to-end data flow from ScreenshotParser handleApply through gameStore, recommendation request, context builder, to Claude's user message -- with system prompt guidance for KDA reasoning.
