  <action>
    Create `prismlab/backend/scripts/generate_training_data.py` — a standalone CLI script that generates JSONL training data by running the full Prismlab pipeline (rules + Claude) across hero/role/matchup combinations.

    **Script structure:**

    ```python
    #!/usr/bin/env python3
    """Generate training data for Ollama fine-tuning.

    Runs Claude API across hero/role/matchup combinations using the actual
    ContextBuilder pipeline. Outputs ChatML-format JSONL for Unsloth.

    Usage:
        python -m scripts.generate_training_data --output training_data.jsonl --limit 100
        python -m scripts.generate_training_data --output training_data.jsonl  # all combinations
    """
    ```

    **Key design decisions:**

    1. **Use ACTUAL ContextBuilder.build()** for prompts (per D-05, research Pitfall 5). NOT simplified templates. Each training example must include the full user message that Claude sees in production.

    2. **Combination generation strategy** (per D-06, research training data strategy):
       - Get all heroes from DataCache (~125)
       - For each hero, determine valid roles (1-5 based on common role assignments). Use a conservative mapping: carry (1), mid (2), offlane (3), soft support (4), hard support (5). Most heroes have 1-2 primary roles.
       - For each hero/role, pick 1-2 playstyles from VALID_PLAYSTYLES
       - For each hero/role/playstyle, generate matchups: pick 2-3 random opponents from the hero pool
       - Side: alternate radiant/dire
       - Lane: derive from role (1 -> safe, 2 -> mid, 3 -> off, 4 -> off, 5 -> safe)
       - Target: ~5,000-10,000 combinations (D-06)

    3. **Hero role mapping**: Create a simple mapping dict. For heroes not in the map, skip (or assign the most common role). The script should be pragmatic, not exhaustive -- training data quality matters more than 100% hero coverage.

    4. **Output format** — ChatML JSONL for Unsloth (per research):
       ```json
       {"messages": [
         {"role": "system", "content": "<SYSTEM_PROMPT>"},
         {"role": "user", "content": "<full context_builder output>"},
         {"role": "assistant", "content": "<LLMRecommendation JSON>"}
       ]}
       ```
       One JSON object per line.

    5. **Execution flow:**
       - Parse CLI args: `--output` (file path), `--limit` (max examples), `--resume` (skip first N), `--delay` (seconds between API calls, default 0.5)
       - Initialize: create async engine, load DataCache, create RulesEngine, ContextBuilder, LLMEngine
       - Generate combinations list
       - Shuffle combinations (for variety if using --limit)
       - For each combination:
         - Build RecommendRequest
         - Run rules.evaluate(request)
         - Build user_message via context_builder.build(request, rules_items, db)
         - Call llm.generate(user_message)
         - If success: write JSONL line to output file
         - If failure: log warning, continue (don't stop on individual failures)
         - Sleep(delay) between calls to respect rate limits
       - Print summary: total generated, failures, elapsed time

    6. **Resume support**: `--resume N` skips the first N combinations. Useful if script is interrupted. The output file is opened in append mode when resuming.

    7. **Progress logging**: Print every 100 examples: "Generated 100/7500 (1.3%), 2 failures, ~$1.20 estimated cost"

    8. **Cost estimation**: Track input/output tokens from `llm.last_usage` (added in Plan 01) and calculate running cost estimate using Haiku batch pricing ($0.50/$2.50 per MTok).

    **Important: This script runs OUTSIDE the FastAPI app.** It needs its own async setup:
    ```python
    async def main():
        # Create engine, session
        async with engine.begin() as conn:
            await conn.run_sync(Base.metadata.create_all)
        async with async_session() as session:
            await data_cache.load(session)
            # ... generate data
    ```

    Run via: `cd prismlab/backend && python -m scripts.generate_training_data --output data/training_data.jsonl --limit 100`
