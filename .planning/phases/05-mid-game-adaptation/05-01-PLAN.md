Add these optional fields to RecommendRequest (all default to None/empty so existing calls still work):

**context_builder.py -- Add mid-game context section to Claude prompt:**
Add a new method _build_midgame_section(self, request: RecommendRequest) -> str that returns empty string if no mid-game fields are present (lane_result is None AND damage_profile is None AND enemy_items_spotted is empty), otherwise builds a section like:

## Mid-Game Update
Lane Result: Won
Damage Taken: Physical 60%, Magical 30%, Pure 10%
Enemy Items Spotted: BKB, Blink Dagger

Call this method in build() and insert the section after the opponent lines but before the item catalog.
Also update the final instruction line: when request.purchased_items is non-empty, change instruction to "Recommend items for REMAINING unpurchased slots only. Focus reasoning on how the game state changes affect remaining item choices."

Map enemy item internal names to display names for the prompt. Simple approach: replace underscores with spaces and title-case each word.

**recommender.py -- Filter purchased items from output:**
In the recommend() method, after the merge step (Step 4) and before _validate_item_ids (Step 5), add a new step that filters out purchased items. Add a private method:

def _filter_purchased(self, phases: list[RecommendPhase], purchased: list[int]) -> list[RecommendPhase]:

That:
1. Converts purchased to a set for O(1) lookup
2. For each phase, filters items whose item_id is NOT in the purchased set
3. Only keeps phases that still have items after filtering
4. Returns the filtered list
Only call this method if request.purchased_items is non-empty.
