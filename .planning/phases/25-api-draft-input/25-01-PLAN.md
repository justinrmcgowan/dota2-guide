**1. Create `prismlab/backend/data/stratz_client.py`** — new Stratz GraphQL client. Per D-01, this is the primary data source. Per D-02, add as a new module (not merged into opendota_client). Use raw httpx POST (no GraphQL library per research findings).

Class `StratzClient`:
- Constructor takes `token: str`. Sets headers: `Authorization: Bearer {token}`, `User-Agent: Prismlab/1.0`, `Content-Type: application/json`.
- `GRAPHQL_URL = "https://api.stratz.com/graphql"`
- Method `async def fetch_player_last_match_id(self, account_id: int) -> int | None`:
  - GraphQL query: `query($steamAccountId: Long!) { player(steamAccountId: $steamAccountId) { matches(request: { take: 1 }) { id } } }`
  - POST to GRAPHQL_URL with json={"query": query, "variables": {"steamAccountId": account_id}}
  - Timeout 10s. Parse response: data.player.matches[0].id or None.
- Method `async def fetch_live_match(self, match_id: int) -> dict | None`:
  - GraphQL query requesting: matchId, gameState, gameTime, radiantScore, direScore, players { steamAccountId, heroId, isRadiant, playerSlot, position, numKills, numDeaths, numAssists, level, networth }
  - POST with variables: {"matchId": match_id}. Timeout 10s.
  - Return data.live.match or None.
- Method `async def fetch_live_match_for_player(self, account_id: int) -> dict | None`:
  - Two-step approach per research: (1) call fetch_player_last_match_id, (2) if match_id found, call fetch_live_match. (3) If result exists and has players, return it. (4) If null (not live), return None.
  - Wrap in try/except httpx.HTTPStatusError and httpx.TimeoutException -- return None on failure.

Use `async with httpx.AsyncClient() as client:` for each request (matches existing opendota_client.py pattern).

**2. Extend `prismlab/backend/data/opendota_client.py`** — add one method:
- `async def fetch_live_match_for_player(self, account_id: int) -> dict | None`:
  - GET `{BASE_URL}/live` with self.params, timeout 15s.
  - Iterate games, iterate game["players"], find player where `account_id` matches.
  - Return the matching game dict or None if not found.
  - Note: OpenDota /live only returns top/popular games (per research Pitfall 1). This is acceptable as fallback.

**3. Extend `prismlab/backend/config.py`** — add `steam_id: str | None = None` to Settings class. Per D-10, this is the default Steam ID from .env that the frontend will pre-fill.

**1. Create `prismlab/backend/api/routes/live_match.py`** — new route module.

Pydantic response models:
- `LiveMatchPlayer(BaseModel)`: account_id: int, hero_id: int, is_radiant: bool, player_slot: int, position: int | None = None (1-5 from Stratz, None from OpenDota)
- `LiveMatchResponse(BaseModel)`: match_id: int, game_state: str, game_time: int, players: list[LiveMatchPlayer], source: str ("stratz" or "opendota"), radiant_score: int = 0, dire_score: int = 0

Create `router = APIRouter()`.

Instantiate clients at module level:
```python
from config import settings
from data.stratz_client import StratzClient
from data.opendota_client import OpenDotaClient

_stratz = StratzClient(settings.stratz_api_token) if settings.stratz_api_token else None
_opendota = OpenDotaClient(settings.opendota_api_key)
```

Endpoint `GET /live-match/{account_id}` (response_model=LiveMatchResponse | None):
- Per D-01: Try Stratz first. If _stratz is not None, call `_stratz.fetch_live_match_for_player(account_id)`. If result is not None, normalize the Stratz response into LiveMatchResponse (source="stratz"). Map Stratz field names: heroId->hero_id, steamAccountId->account_id, isRadiant->is_radiant, playerSlot->player_slot, gameState->game_state, gameTime->game_time. For position, map POSITION_1 through POSITION_5 to 1-5, anything else to None.
- If Stratz fails or returns None, try OpenDota fallback: call `_opendota.fetch_live_match_for_player(account_id)`. Normalize response: map "team": 0 -> is_radiant=True, "team": 1 -> is_radiant=False. team_slot -> player_slot. game_time from response. source="opendota". Position is None for OpenDota (team_slot is connection order, NOT role per research).
- If both fail, return None (FastAPI will return null JSON body).
- Wrap each in try/except Exception with logging -- never let one source's failure prevent trying the other.
