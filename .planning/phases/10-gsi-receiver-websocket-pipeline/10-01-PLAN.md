<objective>
Build the backend GSI receiver module: Pydantic models for parsing Dota 2 GSI JSON payloads, an in-memory state manager singleton, the POST /gsi endpoint, and the GSI config file generator endpoint.

Purpose: This is the data ingestion layer -- Dota 2 sends HTTP POST requests with game state JSON to /gsi at ~2Hz. This plan parses that data into a normalized in-memory state that the WebSocket layer (Plan 02) will broadcast.

Output: Working GSI receiver endpoint, state manager, config generator, and tests proving all D-13 fields are parsed correctly.
</objective>
