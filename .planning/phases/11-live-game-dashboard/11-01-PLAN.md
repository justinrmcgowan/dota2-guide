  <action>
    Create `prismlab/frontend/src/hooks/useAnimatedValue.ts` implementing a requestAnimationFrame-based counting animation hook per D-08 (slot machine style smooth interpolation).

    Signature: `export function useAnimatedValue(target: number, duration?: number): number`
    - Default duration: 300 (milliseconds)
    - Uses `requestAnimationFrame` for smooth 60fps updates
    - Ease-out curve: `1 - (1 - progress) ** 2` for natural deceleration
    - Returns `Math.round()` of interpolated value (gold/GPM are integers)
    - Stores starting value in a ref (NOT state) to avoid re-triggering the effect
    - Cancels previous animation frame on cleanup

    Implementation pattern:
    ```typescript
    export function useAnimatedValue(target: number, duration = 300): number {
      const [display, setDisplay] = useState(target);
      const startValRef = useRef(target);
      const startTimeRef = useRef(0);
      const rafRef = useRef(0);

      useEffect(() => {
        startValRef.current = display;  // capture current display as start
        startTimeRef.current = performance.now();

        const animate = (now: number) => {
          const elapsed = now - startTimeRef.current;
          const progress = Math.min(elapsed / duration, 1);
          const eased = 1 - (1 - progress) ** 2;
          const current = Math.round(startValRef.current + (target - startValRef.current) * eased);
          setDisplay(current);
          if (progress < 1) {
            rafRef.current = requestAnimationFrame(animate);
          }
        };

        rafRef.current = requestAnimationFrame(animate);
        return () => cancelAnimationFrame(rafRef.current);
      }, [target, duration]);

      return display;
    }
    ```

    NOTE: There is a subtle bug in the pattern above -- reading `display` inside useEffect creates a stale closure. Instead, use a separate ref to track the "current display value" that gets updated alongside setDisplay. The ref always holds the latest displayed value, so when a new animation starts it reads from the ref, not the stale `display` variable.

    For tests: use `vi.useFakeTimers()` and mock `requestAnimationFrame` with `vi.stubGlobal`. Use `renderHook` from `@testing-library/react` (already available in the project as `@testing-library/react` is installed). Test that initial render returns the target value, and after rerendering with a new target + advancing timers, the value transitions.
