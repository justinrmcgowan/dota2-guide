    Signature: `export function useGsiSync(heroes: Hero[]): void`

    The hook accepts the heroes array from `useHeroes()` and is mounted once in App.tsx.

    **Implementation details:**

    1. **Subscribe to gsiStore outside render cycle** using `useGsiStore.subscribe((state, prevState) => {...})` in a useEffect. This avoids cascading re-renders (Pitfall 2).

    2. **Use refs for data that changes after mount** (Pitfall 1):
       - `heroesRef = useRef<Hero[]>(heroes)` -- update when heroes prop changes
       - `prevHeroIdRef = useRef<number>(0)` -- track last processed hero_id to avoid redundant calls

    3. **Hero auto-detection (D-01, D-02, GSI-02):**
       - Guard: only process when `state.gsiStatus === "connected"` and `live.hero_id > 0`
       - Guard: only when `live.hero_id !== prevHeroIdRef.current` (changed since last process)
       - Find hero: `heroesRef.current.find(h => h.id === live.hero_id)`
       - If found: call `useGameStore.getState().selectHero(hero)` and update prevHeroIdRef
       - Role suggestion: call `inferRole(hero)` helper function and if result is non-null, call `useGameStore.getState().setRole(result)`
       - If not found: log `console.warn('GSI hero_id ${live.hero_id} not found in heroes list')` and skip

    4. **Role inference helper** (inside the same file, not exported):
       ```typescript
       function inferRole(hero: Hero): number | null {
         const roles = hero.roles.map(r => r.toLowerCase());
         if (roles.includes("carry")) return 1;
         if (roles.includes("nuker") && !roles.includes("support") && !roles.includes("durable")) return 2;
         if (roles.includes("initiator") && roles.includes("durable")) return 3;
         if (roles.includes("support") && roles.includes("disabler")) return 4;
         if (roles.includes("support")) return 5;
         return null; // Ambiguous -- leave unselected, user can choose
       }
       ```
       This is a best-effort heuristic per RESEARCH.md (LOW confidence). For ambiguous heroes, returns null and leaves role unselected (per D-02 fallback).

    5. **Item auto-marking (D-11, D-12, D-13, D-14, GSI-04):**
       - On every gsiStore update where gsiStatus === "connected":
       - Get current recommendations: `useRecommendationStore.getState().data`
       - If no recommendations, skip
       - Call `findPurchasedKeys(live.items_inventory, live.items_backpack, recommendations)` from Plan 01
       - For each key in the result Set that is NOT already in `useRecommendationStore.getState().purchasedItems`, call `togglePurchased(key)` to mark as purchased
       - IMPORTANT: Only ADD items, never remove. If GSI says item is in inventory, mark it. If item was previously marked but is no longer in inventory (sold/dropped), do NOT unmark it -- leave to manual toggle. This prevents flickering on temporary inventory changes.

    6. **Cleanup:** Return unsubscribe function from useEffect.

    **Tests:** Use `vi.fn()` to spy on store actions. Set gsiStore state directly via `useGsiStore.setState()`. Use `renderHook` from `@testing-library/react`. Mock `useGsiStore.subscribe` behavior by triggering state changes in the store.
