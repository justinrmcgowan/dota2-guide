interface ScreenshotParserProps {

State management:

Auto-parse on open (per D-03):
- When `isOpen` transitions to `true` AND `imageData` is not null/empty AND `parsedHeroes` is empty AND `isLoading` is false:
  - Set `isLoading = true`
  - Call `api.parseScreenshot({ image_base64: imageData, media_type: mimeType ?? "image/png" })`
  - On success: `setParsedHeroes(response.heroes)`, `setLatency(response.latency_ms)`, check for `response.error`
  - On error: `setError(err.message)`
  - Finally: `setLoading(false)`
- Use a `useEffect` with `[isOpen, imageData]` dependencies.
- IMPORTANT: Check `imageData && imageData.length > 0` before auto-parsing. When opened via sidebar button with empty imageData, show upload zone instead.

Layout (per D-05):
- **Modal backdrop**: Fixed overlay, `bg-black/60`, click outside to cancel
- **Modal container**: `max-w-3xl w-full mx-auto`, centered vertically, dark bg (`bg-bg-secondary`), rounded-lg, max-h-[85vh] with overflow-y-auto
- **Header**: "Screenshot Parser" title + close X button
- **Screenshot thumbnail** (top): If `imageData`, render `<img src={`data:${mimeType};base64,${imageData}`} />` scaled to fit (max-h-48, object-contain). This is the preview of what was pasted.
- **Loading state**: When `isLoading`, show a skeleton/spinner below the thumbnail with text "Analyzing scoreboard..."
- **Error state**: When `error`, show error message in red text with a "Try Again" button (re-calls parse)
- **Parsed results**: When `parsedHeroes.length > 0`, render each hero as a `<ParsedHeroRow>`. Between rows, a thin divider line.
- **Latency indicator**: Small gray text showing "Parsed in {latencyMs}ms" when available
- **Empty results**: If parsing succeeds but `parsedHeroes` is empty, show "No heroes detected in screenshot"
- **Footer buttons** (per D-08):
  - "Cancel" button (gray, left): calls `closeModal()`
  - "Apply to Build" button (cyan accent #00d4ff, right): calls `handleApply()`. Disabled when `parsedHeroes.length === 0` or `isLoading`.

`handleApply()` function (per D-13, D-14, D-15, D-16):
1. **Set opponents (D-14)**: For each parsedHero (up to 5):
   - Find the matching Hero object: `heroes.find(h => h.id === parsedHero.hero_id)`

   ```typescript
   const allItems = parsedHeroes.flatMap(h => h.items.map(i => i.internal_name));
   const uniqueItems = [...new Set(allItems)];
   useGameStore.getState().setEnemyItemsSpotted(uniqueItems);
   ```

4. **Trigger recommendation refresh (D-16)**: Use `useRecommendation()` hook directly inside ScreenshotParser to get `recommend`. Call `recommend()` -- this is user-initiated so it bypasses auto-refresh cooldown.
5. **Close modal**: `closeModal()`

  - Dashed border area with "Drag & drop a screenshot here, or click to browse"

  - On file select or drop: read file via FileReader.readAsDataURL, strip `data:...;base64,` prefix, then call `useScreenshotStore.getState().openModal(base64, mimeType)` which will re-trigger the auto-parse useEffect.

**ItemEditPicker integration**:
