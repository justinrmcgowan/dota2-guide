**2. Create `prismlab/backend/engine/prompts/system_prompt.py`** — this is the heart of the app per CLAUDE.md.

Define a constant `SYSTEM_PROMPT: str` that is the full system prompt text. It MUST be at least 2048 tokens to enable prompt caching (Sonnet 4.6 minimum). The prompt must contain:

**Section 1: Role Definition**
```
You are an elite Dota 2 item advisor with 8,000+ MMR analytical expertise. You analyze hero matchups, game state, and item synergies to recommend optimal itemization.

Your recommendations must be:
- SPECIFIC: Every reasoning MUST name at least one enemy hero AND reference a specific ability, mechanic, or interaction. Never give generic advice like "this is a strong item."
- ANALYTICAL: Use data-driven language. Reference win rates, damage numbers, timing windows, gold efficiency. Think like a coach reviewing a replay, not a friend giving casual advice.
- PHASED: Recommend items in game phases (starting, laning, core, late_game). Each phase has different gold budgets and strategic goals.
- COMPLEMENTARY: You receive a list of "already recommended" items from a rules engine. Do NOT duplicate these. Build around them.
```

**Section 2: Game Knowledge Principles**
Include 8-10 key Dota itemization principles that guide reasoning:
- Starting items: prioritize regen and stats for lane sustain
- Laning phase: cost-efficient items under 2000g that win the lane (Wraith Band, Bracer, Magic Wand upgrade, Boots)
- Core items: 2000-5000g items that define the hero's power spike timing
- Late game: luxury items 5000g+ that scale into late
- Situational items: items bought ONLY in response to specific enemy threats (BKB, MKB, Manta for dispel, Linken's for targeted spells)
- Gold efficiency: support items must be cheap. Pos 5 rarely exceeds 3000g per item
- Build path matters: components should be useful individually (Bracer components all give stats)
- Timing windows: some heroes peak early (Huskar), others need items to come online (Spectre). Recommend accordingly.

**Section 3: Output Constraints**
```
CRITICAL RULES:
1. You may ONLY recommend items from the provided item list. Do not reference items not in the list.
2. Every "reasoning" field MUST mention at least one enemy hero by name.
3. Every "reasoning" field MUST reference a specific ability, mechanic, or stat interaction.
4. Use integer item_id values from the provided item list, not free-text item names.
5. Do NOT recommend items already in the "already_recommended" list.
6. Recommend 2-4 items per game phase. Not every phase needs items.
7. "conditions" field is for situational items only. Format: "If [condition] -> [item reason]"
8. "priority" must be one of: "core" (buy every game), "situational" (buy in response to threat), "luxury" (buy if ahead)
```

**Section 4: Few-Shot Example (Good)**
Include one complete example showing a Pos 1 Anti-Mage vs Zeus + Crystal Maiden lane:
- Show good reasoning: "Against Zeus's Arc Lightning and Crystal Maiden's Crystal Nova, the combined magical burst output makes early Magic Wand critical. Zeus alone generates 5-8 charges per minute from constant Q spam, making this a 200-gold investment that frequently saves 200+ effective HP."
- Show phased items with timing context
- Show one situational item with conditions

**Section 5: Few-Shot Example (Bad — do not emulate)**
Show one example of bad reasoning to avoid: "BKB is a good item because it gives magic immunity." Label this as BAD and explain why: no enemy hero named, no ability referenced, no matchup context.

The total system prompt should be 2500-3500 tokens (comfortably above the 2048 cache minimum, but not bloated). This prompt is cached so it is sent once and reused across requests.

**1. Create `prismlab/backend/engine/llm.py`** with class `LLMEngine`:

```python
import json
import logging
from anthropic import AsyncAnthropic, APITimeoutError, APIConnectionError, APIStatusError
from engine.schemas import LLMRecommendation, LLM_OUTPUT_SCHEMA
from engine.prompts.system_prompt import SYSTEM_PROMPT
from config import settings

logger = logging.getLogger(__name__)

class LLMEngine:
    MODEL = "claude-sonnet-4-6-20250514"
    MAX_TOKENS = 2000
    TEMPERATURE = 0.3
    TIMEOUT_SECONDS = 10.0

    def __init__(self):
        self.client = AsyncAnthropic(api_key=settings.anthropic_api_key)

    async def generate(self, user_message: str) -> LLMRecommendation | None:
        """Call Claude API with structured output and return validated recommendation.

        Uses:
        - output_config.format with JSON schema (GA, not beta)
        - Prompt caching on system prompt (cache_control ephemeral)
        - 10-second hard timeout via client.with_options()
        - max_retries=0 (no retries within timeout budget)

        Returns LLMRecommendation on success, None on any failure (triggers fallback).
        """
        try:
            response = await self.client.with_options(
                timeout=self.TIMEOUT_SECONDS,
                max_retries=0,
            ).messages.create(
                model=self.MODEL,
                max_tokens=self.MAX_TOKENS,
                temperature=self.TEMPERATURE,
                system=[
                    {
                        "type": "text",
                        "text": SYSTEM_PROMPT,
                        "cache_control": {"type": "ephemeral"},
                    }
                ],
                messages=[{"role": "user", "content": user_message}],
                output_config={
                    "format": {
                        "type": "json_schema",
                        "schema": LLM_OUTPUT_SCHEMA,
                    }
                },
            )
            data = json.loads(response.content[0].text)
            return LLMRecommendation.model_validate(data)
        except APITimeoutError:
            logger.warning("Claude API timeout after %.1fs", self.TIMEOUT_SECONDS)
            return None
        except APIConnectionError:
            logger.warning("Claude API connection error")
            return None
        except APIStatusError as e:
            logger.error("Claude API error: %s %s", e.status_code, e.message)
            return None
        except Exception as e:
            logger.exception("Unexpected error calling Claude API: %s", e)
            return None
```

Key implementation details:
- `system` parameter is a LIST of content blocks (not a string) to enable `cache_control`
- `cache_control: {"type": "ephemeral"}` enables 5-minute prompt caching on Sonnet 4.6
- `output_config.format` is the GA path for structured outputs (NOT `output_format`, NOT `beta.messages.parse()`)
- `max_retries=0` because we have a hard 10-second timeout budget and retries would consume it
- Every failure path returns None (the orchestrator in Plan 03 treats None as fallback trigger)
- JSON is parsed from `response.content[0].text` and validated through `LLMRecommendation.model_validate()`
