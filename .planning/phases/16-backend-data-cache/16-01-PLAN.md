  <action>
Create `prismlab/backend/data/cache.py` with:

1. **HeroCached frozen dataclass** -- mirror all Hero model fields except `updated_at`. Use `tuple[str, ...]` for `roles` (not list, for immutability). All fields match Hero column names exactly so `HeroCached(**row_dict)` works for API serialization.

2. **ItemCached frozen dataclass** -- mirror all Item model fields except `updated_at`. Use `tuple[str, ...] | None` for `tags` and `tuple | None` for `components`. Keep `bonuses` as `dict | None` (read-only by convention, frozen dataclass prevents reassignment).

3. **DataCache class** with:
   - `_heroes: dict[int, HeroCached]` (keyed by hero ID)
   - `_items: dict[int, ItemCached]` (keyed by item ID)
   - `_hero_name_to_id: dict[str, int]` (localized_name -> ID)
   - `_item_name_to_id: dict[str, int]` (internal_name -> ID)
   - `_initialized: bool` (starts False)

4. **`async def load(self, db: AsyncSession)`** -- query all heroes and items, build frozen dataclasses, then atomically swap all four dicts at once (per D-02). Convert `roles` list to tuple, `tags` list to tuple, `components` list to tuple. Set `_initialized = True`.

5. **`async def refresh(self, db: AsyncSession)`** -- alias for `load()`. The atomic swap pattern means this is safe to call while reads are happening (per D-02, single-threaded async).

6. **Synchronous lookup methods (zero DB, pure dict reads):**
   - `get_hero(hero_id: int) -> HeroCached | None`
   - `get_all_heroes() -> list[HeroCached]` -- sorted by localized_name
   - `get_all_items() -> list[ItemCached]` -- sorted by name
   - `get_item(item_id: int) -> ItemCached | None`
   - `hero_name_to_id(name: str) -> int | None` -- by localized_name
   - `item_name_to_id(internal_name: str) -> int | None` -- by internal_name
   - `hero_id_to_name(hero_id: int) -> str` -- returns localized_name, falls back to "the enemy"
   - `get_item_name_map() -> dict[int, str]` -- {item_id: item.name} for all items (used by context_builder._extract_top_items)
   - `get_item_validation_map() -> dict[int, tuple[int | None, str]]` -- {item_id: (cost, internal_name)} for recommender._validate_item_ids
   - `get_relevant_items(role: int) -> list[dict]` -- replicate matchup_service.get_relevant_items logic: exclude recipes, neutrals, zero-cost; apply role budget (max 10000g for roles 1-3, 5500g for roles 4-5); return list of `{"id", "name", "cost"}` dicts sorted by cost, capped at 50. Note: this method does NOT take hero_id -- the current get_relevant_items ignores hero_id too.
   - `get_neutral_items_by_tier() -> dict[int, list[dict]]` -- replicate matchup_service.get_neutral_items_by_tier: filter is_neutral and tier is not None; group by tier; return `{tier: [{id, name, internal_name, active_desc}]}`.

7. **Module-level singleton:** `data_cache = DataCache()` at bottom of file.

Use `from __future__ import annotations` for cleaner type hints. Import AsyncSession from sqlalchemy. Import select from sqlalchemy. Import Hero and Item from data.models.
