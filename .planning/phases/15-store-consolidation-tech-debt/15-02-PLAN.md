**Hook signature:** `export function useGameIntelligence(heroes: Hero[]): void`

**Internal structure** (per D-03 ordering):

1. **Refs** — Combine all refs from both hooks:
   - `heroesRef` (from useGsiSync)
   - `prevHeroIdRef` (from useGsiSync)
   - `prevStateRef` (from useAutoRefresh)
   - `firedPhasesRef` (from useAutoRefresh)
   - `cooldownEndRef` (from useAutoRefresh)
   - `queuedEventRef` (from useAutoRefresh)
   - `laneAutoDetectedRef` (from useAutoRefresh)
   - `prevGsiStatusRef` (from useAutoRefresh)

2. **`inferRole()` function** — Copy exactly from useGsiSync (local helper, not exported).

3. **`suggestPlaystyle()` function** — NEW (per D-05, D-06):
   ```typescript
   function suggestPlaystyle(heroId: number, role: number): string {
     const key = `${heroId}-${role}`;
     const mapped = HERO_PLAYSTYLE_MAP[key];
     if (mapped) return mapped;
     // Fallback: first valid playstyle for this role
     return PLAYSTYLE_OPTIONS[role]?.[0] ?? "balanced";
   }
   ```

4. **`fireRefresh()` function** — Copy exactly from useAutoRefresh.

5. **`useEffect` to keep heroesRef in sync** — Copy from useGsiSync.

6. **FIRST `useEffect` — gsiStore subscription for hero detection + item marking + lane detection + event detection + cooldown** (per D-03 ordering):

   This is the main gsiStore.subscribe callback. It processes in this deterministic order:

   a. Guard: only when `gsiStatus === "connected"` and `liveState` exists
   b. Handle GSI reconnect (sync prev state, same as useAutoRefresh)
   c. **Hero auto-detection** (from useGsiSync): When hero_id changes, find hero, call `selectHero()`, call `inferRole()`, then call `setRole()`. **NEW: After setRole, call `suggestPlaystyle(heroId, role)` and set it via `gameStore.getState().setPlaystyle(playstyle)`** (per D-06). This MUST happen in one synchronous block AFTER setRole. Per Pitfall 15, batch the writes: call `setRole(role)` first (which may clear playstyle), then immediately `setPlaystyle(suggested)` so the user never sees a null playstyle flash.
   d. **Item auto-marking** (from useGsiSync): `findPurchasedKeys()` then `togglePurchased()` for new items
   e. Guard: only detect events during active game state, with existing recommendations, not loading
   f. **Lane auto-detection** (from useAutoRefresh): at 10:00 game clock, `detectLaneResult()`
   g. **Event detection** (from useAutoRefresh): `detectTriggers()`, cooldown check, queue or fire

   The key insight: hero detection MUST complete before event detection so `game.role` is available for lane benchmarks. This ordering is guaranteed because they are sequential in the same callback.

   Include the **1Hz `setInterval`** (per D-04) inside this same useEffect, same as useAutoRefresh. It handles:
   - `refreshStore.tick(now)` countdown
   - Queued event firing when cooldown expires

   Return cleanup: `unsubscribe()` + `clearInterval(interval)`.

7. **SECOND `useEffect` — recommendationStore subscription** (separate per D-02):

   Copy the manual-recommend-completes detection from useAutoRefresh exactly:
   - Track `wasLoading` transition
   - Reset cooldown on completion
   - Reset gold baseline
   - Clear queue

   Return cleanup: `unsubscribe()`.

**Per D-07:** User can always override playstyle manually. The auto-suggest only fires when a NEW hero is detected (hero_id changes). It does NOT re-set playstyle on every GSI tick. The `prevHeroIdRef` guard ensures this.
